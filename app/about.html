About

The Docker Registry Frontend utilizes the Docker Registry API to show and manipulate the content of any private Docker registry.

The source code can be found on Github but if you just want to run your own copy of the software, I suggest you run it in its own Docker container with an automatically generated image from The Docker Registry Hub.

Tools used

The Docker Registry Frontend is built with these tools:

AngularJS

AngularJS sits at the core of all development. It is used to separate content from logic through mechanisms like two-way data-binding and a clear MVC pattern.

Checkout AngularJS

YEOMAN

YEOMAN was used to setup the project with all the initial config files. It helps to be productive very quickly.

Checkout YEOMAN

Grunt

The Grunt task runner is used to serve the website during development, to test the code, and to build (e.g. minification) the code for deployment.

Checkout Grunt

Bower

The Bower package manager is used to manage all the dependencies of the code.

Checkout Bower
